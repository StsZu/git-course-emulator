::Staged-стан::
Що означає стан staged? {
=Файл підготовлений до збереження, але ще не зафіксований у локальній базі Git
~Файл остаточно видалений з проекту
~Файл автоматично завантажений на GitHub
#### Staged — коли файл «підготовлено до збереження, але він ще не збережений» у локальній базі даних Git.
}

::Проблеми ручного контролю версій::
Яка проблема командної роботи без Git? {
=Майже неможливо зшивати зміни кількох людей в один файл без конфліктів
~Файли не можна копіювати взагалі
~Кожен розробник автоматично бачить усі зміни в реальному часі
#### Без системи контролю версій «працювати в команді дуже важко» — потрібен хтось, хто зшиває майстер-версію і відстежує конфлікти.
}

::Гілка (branch)::
Навіщо потрібні гілки в Git? {
=Для ізольованої розробки та тестування без впливу на основний код
~Щоб видаляти історію комітів
~Щоб заборонити командну роботу
#### Гілки «дозволяють створювати окремі середовища для безпечного тестування» — зміни не впливають на основний проект до merge.
}

::Merge::
Що таке merge у контексті Git? {
=Злиття гілок після завершення розробки та тестування
~Видалення всіх файлів проекту
~Перейменування репозиторію
#### Після тестування в гілці можна «організувати злиття гілок (merge)» — об'єднати зміни з основною гілкою.
}

::Git проти GitHub::
Чим принципово відрізняється GitHub від Git? {
=Git — локальне ПЗ; GitHub — хмарна платформа/сервіс для зберігання та співпраці
~Це два назви одного продукту
~GitHub встановлюється лише на Windows
#### Git встановлюється локально, GitHub — «сервіс, платформа», що зберігає в хмарі історію змін і дозволяє співпрацювати.
}

::Відстеження текстових файлів::
Для яких файлів Git надає найдетальніше відстеження змін? {
=Для файлів, що містять текст (код, документи, скрипти)
~Лише для зображень у форматі PNG
~Лише для зашифрованих бінарних файлів без змін
#### Git найкраще відстежує текстові файли; з іншими типами теж працює, але «не надає такого детального відстеження змін».
}

::Копія історії в команді::
Що має кожен член команди при роботі з Git? {
=Локальну відзеркалену копію всієї історії змін проекту
~Лише останній файл без історії
~Доступ лише до README
#### У кожного в команді є «локальна відзеркалена копія всієї історії змін» — це додатковий рівень безпеки даних.
}

::Підхід до навчання Git::
Який підхід до вивчення Git рекомендує автор? {
=Поступово, одразу тренуючись на практиці
~Вивчити лише теорію без команд
~За один день без повторення основ
#### «Якщо вивчати його поступово і відразу тренуватися на практиці, він стане вашим інструментом на кожен день».
}
