  =Документація послідовна, детальна, зі скріншотами та покроковими інструкціями для новачків
  ~Бо офіційна документація доступна лише українською мовою
  ~Бо конспекти курсу заборонено використовувати після реєстрації на GitHub

::Мови інтерфейсу документації::
Яка ситуація з українською мовою в документації GitHub згідно з уроком? {
  =У Get started є англійська, іспанська, російська, французька, але української немає
  ~Українська — мова за замовчуванням для всіх розділів
  ~Документація доступна лише японською та німецькою
  #### Відсутність української не блокує навчання: структура гайдів зрозуміла, а термінологія вже знайома з курсу.
}

::Унікальність назви::
Яке правило діє для назви нового репозиторію в межах особистого акаунту? {
  =Назва має бути унікальною в межах вашого профілю
  ~Назва має збігатися з іменем кожного учасника команди
  ~Назва повинна бути унікальною для всього інтернету, як домен .com
  #### Два ваші репозиторії не можуть мати однакову назву, але інший користувач може використати те саме ім'я в своєму профілі.
}

::Публічний репозиторій::
Що означає публічний репозиторій на GitHub? {
  =Будь-хто може отримати доступ до проєкту й коду, навіть без акаунту GitHub
  ~Доступ мають лише користувачі з вашим паролем
  ~Код видно тільки вам і службі підтримки GitHub
  #### Public — свідомий вибір для портфоліо та відкритого поширення напрацювань.
}

::Приватний репозиторій::
Коли логічно обрати приватний репозиторій? {
  =Коли код не потрібно показувати всій спільноті, але ви все одно хочете хмарне сховище
  ~Коли потрібно заборонити git commit усередині проєкту
  ~Коли репозиторій не може містити README
  #### Private дає контроль видимості, зберігаючи переваги віддаленого сховища.
}

::README при створенні::
Навіщо додавати README під час створення репозиторію? {
  =Щоб одразу мати файл опису проєкту, який GitHub покаже на головній сторінці репозиторію
  ~Щоб автоматично згенерувати SSH-ключ
  ~Щоб замінити .git папку локального клону
  #### README — перше, що бачить відвідувач репозиторію; його зручно створити на старті.
}

::SSH-ключі в документації::
Для чого в документації GitHub описують SSH-ключі? {
  =Для безпечного обміну інформацією між локальною машиною та GitHub без постійного введення пароля
  ~Для шифрування README у форматі PDF
  ~Для видалення історії комітів після публікації
  #### SSH — механізм довіри між вашим комп'ютером і акаунтом GitHub.
}

::Власник репозиторію::
Що бачить індивідуальний користувач у полі Owner під час створення репозиторію? {
  =Зазвичай лише себе; у командах можуть з'являтися інші опції
  ~Завжди список усіх користувачів GitHub
  ~Лише організацію Microsoft
  #### Для особистого акаунту ви — власник; командні сценарії розширюють вибір owner.
}

::Профільний репозиторій username::
Які умови обов'язкові для репозиторію, що оформлює головну сторінку профілю? {
  =Публічний репозиторій з назвою, що збігається з username, і README-файлом
  ~Приватний репозиторій із довільною назвою без README
  ~Будь-який форк чужого проєкту зі зірочкою
  #### Це спеціальна конвенція GitHub: username/username + README = вітрина профілю.
}
