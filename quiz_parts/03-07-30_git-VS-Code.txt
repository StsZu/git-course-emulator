::Копія папки та .git::
Чому в VS Code-уроці видаляють папку .git після копіювання my_project? {
=Копія несе стару історію комітів; для чистої практики потрібен проект без попереднього репозиторію
~VS Code не відкриває папки, де є приховані файли
~Папка .git блокує встановлення Python-розширень
#### .git копіюється разом із папкою — без видалення ви продовжите чужу історію, а не почнете з нуля.
}

::Source Control::
Яка панель VS Code відповідає за роботу з Git у цьому відео? {
=Source Control (Контроль версій) — аналог git status, add і commit у GUI
~Explorer — лише для перегляду зображень
~Extensions — для встановлення лише тем оформлення
#### Source Control зводить Git-цикл у візуальний інтерфейс: зміни, stage, commit, гілки.
}

::Ініціалізація в VS Code::
Що робить кнопка «Initialize Repository» у VS Code? {
=Створює локальний Git-репозиторій (аналог git init) у відкритій папці
~Завантажує проект з GitHub без вашого облікового запису
~Автоматично публікує код у Main на сервері
#### Ініціалізація через GUI робить те саме, що git init — додає .git і вмикає контроль версій.
}

::Позначка U у Explorer::
Що означає літера U біля файлу після ініціалізації репозиторію? {
=Untracked — файл ще не додано до відстеження Git
~Uploaded — файл уже на GitHub
~Unlocked — файл захищений від редагування
#### U — візуальний аналог untracked у git status; далі файл stage-ять через «+».
}

::Кнопка «+» (Stage)::
Що відповідає натисканню «+» біля файлу в Source Control? {
=git add — передача файлу в staging area (секція Staged Changes)
~git rm — безповоротне видалення з диска
~git push — відправка на віддалений сервер
#### Плюс у GUI = git add: файл переходить із Changes у Staged Changes.
}

::Файл .gitignore::
Навіщо створюють .gitignore у практиці VS Code? {
=Щоб перелічити файли, які Git не повинен відстежувати
~Щоб прискорити компіляцію Python-коду в 10 разів
~Щоб замінити папку .git повністю
#### .gitignore — «чорний список» для Git: локальні/секретні/тимчасові файли лишаються поза історією.
}

::Commit All::
Що показує експеримент із Commit All без явного stage? {
=VS Code може закомітити зміни, навіть якщо ви не натискали «+» — stage відбувається непомітно
~Commit All працює лише після git push
~Commit All видаляє всі unstaged файли
#### GUI інколи об'єднує кроки: корисно знати, що під капотом все одно add + commit.
}
