::Призначення README::
Яку головну роль відіграє README.md у репозиторії GitHub згідно з уроком? {
  =Це вітрина проєкту: опис, інструкції, приклади та презентація ідей автора
  ~Це технічний файл, який замінює папку .git локально
  ~Це прихований конфіг для SSH-ключів
  #### README — перше, що бачить відвідувач; саме тут формується перше враження про проєкт.
}

::Markdown у README::
У якому форматі зазвичай пишуть README на GitHub? {
  =У форматі Markdown з можливістю заголовків, списків, посилань і зображень
  ~У бінарному форматі PDF, який GitHub конвертує автоматично
  ~Лише у вигляді HTML без підтримки простого тексту
  #### Markdown дає читабельний синтаксис і водночас рендериться в охайний веб-вигляд.
}

::Fork як спосіб вивчення README::

::Preview vs Edit::
Чим корисні режими Preview і Edit при роботі з README? {
  =Preview показує готовий вигляд сторінки, Edit — сирий Markdown і прийоми форматування
  ~Preview видаляє всі зображення, Edit додає їх автоматично
  ~Preview доступний лише власнику, Edit — усім користувачам інтернету
  #### Поєднання режимів допомагає зрозуміти, як синтаксис перетворюється на візуальний результат.
}
