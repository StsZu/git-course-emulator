=== QUIZ (GIFT) ===

::Що таке Fork::
Як урок визначає fork репозиторію на GitHub? {
  =Віддалена копія віддаленого репозиторію іншого автора у вашому профілі
  ~Тимчасовий перегляд файлів без збереження копії
  ~Автоматичне злиття ваших змін в оригінал без підтвердження
  #### Fork створює повноцінну незалежну копію з власною історією комітів у вашому акаунті.
}

::Незалежність після fork::
Що відбувається зі змінами, які ви робите у своєму fork? {
  =Вони залишаються у вашій копії і не змінюють оригінальний репозиторій автоматично
  ~Вони миттєво з'являються в репозиторії автора
  ~Вони видаляють оригінальний репозиторій з GitHub
  #### Fork розділяє простори: ви експериментуєте у своїй копії, автор контролює оригінал.
}

::Хто не може зробити fork::
Кому недоступна кнопка Fork для репозиторію згідно з демонстрацією в уроку? {
  =Автору-власнику репозиторію та користувачам без акаунту GitHub
  ~Лише користувачам з приватними репозиторіями
  ~Будь-якому користувачу після першого коміту
  #### Власник не форкає власний проєкт; для додавання копії до профілю потрібен акаунт.
}

::Історія комітів у fork::
Що зберігається в fork разом із файлами проєкту? {
  =Повна історія комітів, включно з попередніми змінами автора
  ~Лише останній стан файлів без історії
  ~Тільки README без коду
  #### Fork копіює не знімок, а весь репозиторій — з комітами, гілками та метаданими.
}

::Аналогія з гілкою::
Чим fork схожий на створення нової гілки за словами автора? {
  =Це відокремлення для роботи над змінами, але без впливу на оригінал
  ~Це повне видалення основної гілки main
  ~Це автоматичний merge у main автора
  #### Як і гілка, fork дає ізольований простір; на відміну від гілки — у вашому власному репозиторії.
}

::Видимість fork для автора::
Чи бачить автор оригіналу зміни у вашому fork без Pull Request? {
  =Ні, зміни у fork не відображаються в оригіналі; автор може бачити факт fork у insights
  ~Так, кожен коміт у fork одразу з'являється в оригіналі
  ~Так, але лише якщо змінено README
  #### Fork — окремий репозиторій; повернення змін до автора відбувається через Pull Request.
}

::Зміна видимості репозиторію::
Що попереджає GitHub при переведенні публічного репозиторію в приватний? {
  =Можливу втрату зірочок, підписників і необхідність кілька разів підтвердити рішення
  ~Автоматичне видалення всіх форків без попередження
  ~Неможливість повернути репозиторій у публічний стан
  #### Зміна видимості — серйозна дія; GitHub навмисно ускладнює її підтвердження.
}

::Навіщо робити fork::
Яка практична мета fork у демонстрації з git-курс репозиторієм? {
  =Отримати власну копію для експериментів і подальшої пропозиції змін автору
  ~Заблокувати оригінальний репозиторій для інших користувачів
  ~Замінити автора як власника проєкту
  #### Fork — стартовий крок контрибуції: спочатку копія, потім зміни, далі Pull Request.
}

Що потрібно зробити після редагування README у власному fork? {
  =Закомітити зміни, щоб вони збереглися в історії вашої копії
  ~Одразу написати автору в особисті повідомлення без коміту
  ~Видалити .git і створити новий репозиторій

}

До якого наступного кроку підводить урок після змін у fork? {
  =До створення Pull Request, щоб запропонувати зміни автору оригіналу
  ~До видалення оригінального репозиторію
  ~До зміни username на GitHub
  #### Fork сам по собі не повертає зміни автору — для цього існує Pull Request.
}
