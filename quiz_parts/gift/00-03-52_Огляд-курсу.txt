::Git простими словами::
Коли в огляді курсу говорять про Git «простими словами», про що йдеться? {
=Про контроль версій файлів
~Про хмарне сховище фотографій
~Про операційну систему Windows
#### У відео Git пояснюється як системний контроль версій — на відміну від ручного створення файлів на кшталт Final Draft v4, v5, v6.
}

::Недоліки ручного контролю версій::
Який ризик залишається при ручному менеджменті версій файлів? {
=Ризик втратити прогрес через людський фактор
~Повна автоматична синхронізація між усіма учасниками команди
~Неможливість створити більше однієї версії файлу
#### Навіть коли ми зберігаємо Final Draft v4, v5, v6, «все ще залишаються ризики втратити прогрес» — через людський фактор.
}

::Git як локальне ПЗ::
Де встановлюється Git згідно з оглядом курсу? {
=Локально на комп'ютер користувача
~Лише на серверах GitHub
~Тільки в хмарному браузері
#### Git — це програмне забезпечення, яке «встановлюється локально на наш комп'ютер» і веде реєстр змін у проекті.
}

::Git і GitHub — різні інструменти::
Як у відео пояснюється різниця між Git і GitHub? {
=Це абсолютно різні речі з різною метою, але вони добре поєднуються
~Це одне й те саме програмне забезпечення
~GitHub замінює потребу в локальному Git
#### Автор підкреслює: «Git та GitHub — це абсолютно різні речі», але «дуже добре поєднуються» у реальній роботі.
}

::Роль GitHub::
Що робить GitHub у схемі роботи з проектом? {
=Хостить код і зберігає найнадійнішу версію проекту в хмарі
~Видаляє локальні копії проекту з комп'ютерів
~Замінює потребу в комітах
#### GitHub «буде хостити ці зміни», зберігати проект і ставати «зосередженням останньої найбільш надійної версії».
}

::Перевага Git як «збереження в грі»::
До чого автор порівнює можливість відновлення версій у Git? {
=До збереження під час проходження комп'ютерної гри
~До автоматичного резервного копіювання в хмару щогодини
~До роботи лише з одним файлом без історії
#### Найбільша перевага Git — можливість «легко відновити попередні версії», як «збереження під час проходження гри».
}

::Для кого призначений курс::
Для кого автор розробив цей курс? {
=Для початківців, які вивчають Git і GitHub з нуля
~Для senior-розробників із глибоким досвідом Git
~Лише для адміністраторів Linux-серверів
#### Курс «розробив для початківців»: якщо вже є глибоке розуміння Git, матеріал буде нецікавим.
}

::Рекомендація щодо часу навчання::
Яку пораду дає автор щодо часу на проходження курсу? {
=Виділити значно більше часу, ніж тривалість самих відео
~Достатньо лише переглянути відео без практики
~Вивчити все за один вечір без вправ
#### Час на курс «має бути значно більший, ніж час власне цього курсу», бо потрібні практичні вправи й засвоєння нового матеріалу.
}

::Матеріали курсу на GitHub::
Які дві основні папки з матеріалами є в репозиторії курсу? {
=Ноутбуки (конспекти) і ресурси
~Відео та сертифікати
~Лише тести без конспектів
#### У репозиторії курсу є «папка ноутбуків» (конспекти) і «папка ресурсів» з інструментами, документацією та тренажерами.
}

::GitHub як портфоліо::
Яку додаткову роль GitHub підкреслює автор у огляді курсу? {
=Платформу для демонстрації навичок і портфоліо роботодавцям
~Заміну всіх соцмереж для IT-спеціалістів
~Інструмент лише для зберігання паролів
#### GitHub «став дуже популярним рішенням для розміщення портфоліо» і демонстрації навичок потенційним роботодавцям.
}
