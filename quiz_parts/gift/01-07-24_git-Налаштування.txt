::Перевірка версії Git::
Якою командою перевіряють встановлену версію Git? {
=git --version
~git --install
~bash --version
#### Команда git --version показує поточну версію ПЗ; у відео це 2.47, але новіша версія теж підходить.
}

::Глобальне налаштування імені::
Як встановити ім'я користувача глобально для всіх репозиторіїв? {
=git config --global user.name "Ваше Ім'я"
~git init --global username
~git status --name
#### git config --global user.name встановлює ім'я «справедливим для всіх репозиторіїв» на цьому комп'ютері.
}

::Навіщо вказувати email у Git::
Чому потрібно налаштувати user.email? {
=Щоб Git асоціював коміти з конкретним користувачем, особливо в командній роботі
~Щоб отримати лист з GitHub щодня
~Щоб замінити пароль від облікового запису
#### Git «буде асоціювати користувача зі мною» при кожній зміні — це важливо, коли працює кілька людей.
}

::Вставка в Git Bash::
Яку комбінацію використовують для вставки в Git Bash замість Ctrl+V? {
=Shift+Insert
~Ctrl+V
~Alt+V
#### У Git Bash для вставки потрібно Shift+Insert, а не звичний Ctrl+V.
}

::Перегляд усіх налаштувань::
Якою командою переглянути всі налаштування Git? {
=git config --list
~git settings --all
~ls --config
#### git config --list показує перелік налаштувань, серед яких мають бути user.name і user.email.
}

::Ініціалізація репозиторію::
Якою командою Git починає відстежувати папку проекту? {
=git init
~git start
~git create repo
#### git init змушує Git «почати працювати і відстежувати зміни» у папці my-project.
}

::Папка .git::
Що з'являється після git init? {
=Прихована папка .git з метаданими та історією проекту
~Публічний файл password.txt
~Автоматичний акаунт на GitHub
#### Після init з'являється прихована директорія .git — «дуже важливе місце», де Git зберігає всі зміни.
}

::Перегляд прихованих файлів::
Як побачити приховану папку .git у терміналі? {
=ls -a
~ls --hidden-only
~cat .git
#### Звичайний ls не показує .git; потрібно ls -a, щоб «побачити всі папки, в тому числі приховані».
}

::Чи можна редагувати .git вручну::
Що автор каже про вміст папки .git? {
=Ніколи не змінювати її структуру та документи вручну
~Редагувати config щодня для прискорення Git
~Видаляти .git після кожного коміту
#### «У жодному разі ми нічого тут ніколи не змінюємо» — це службова зона Git, до якої не повертаються.
}

::git status::
Яку команду автор називає першою при роботі з Git? {
=git status
~git push
~git clone
#### «Майже кожного разу, коли ми будемо працювати з Git, ми будемо починати з git status» — щоб знати поточний стан.
}
