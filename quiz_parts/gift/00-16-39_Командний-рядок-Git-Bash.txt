::Офіційний сайт Git::
Який офіційний веб-сайт Git згадується у відео? {
=git-scm.com
~github.com
~gitbash.org
#### Автор шукає Git у пошуку і переходить на офіційний сайт git-scm.com — SCM означає Source Code Management.
}

::SCM у назві сайту::
Що означає SCM у контексті git-scm.com? {
=Source Code Management — управління вихідним кодом
~System Control Module — модуль контролю системи
~Secure Cloud Memory — захищена хмарна пам'ять
#### SCM розшифровується як Source Code Management — по суті інша назва Version Control System.
}

::Встановлення Git на Windows::
Яку версію Git завантажує автор у відео для Windows? {
=2.47.1
~1.0.0
~3.99.0
#### Під час завантаження згадується версія 2.47.1; у вас може бути новіша, і це не критично.
}

::Налаштування під час інсталяції::
Яку стратегію встановлення Git Bash рекомендує автор? {
=Здебільшого обирати налаштування за замовчуванням
~Змінити всі параметри вручну, ігноруючи рекомендації
~Встановити лише Git без Bash
#### Процес встановлення «буде нескладний» — «здебільшого обираємо всі налаштування за замовчуванням».
}

::Головна гілка за замовчуванням::
Яку назву головної гілки обирає автор замість Master? {
=Main
~Develop
~Trunk
#### Під час інсталяції пропонується Master, але автор «здебільшого використовує Main» і обирає відповідний варіант.
}

::Редактор за замовчуванням::
Який редактор коду за замовчуванням обирає автор під час встановлення? {
=Vim
~Блокнот як єдиний можливий варіант
~GitHub Desktop
#### Пропонується vim, і автор вважає, що «є сенс використовувати надалі», хоча можна обрати Notepad++ або VS Code.
}

::Середовище MINGW64::
Що означає MINGW64 у рядку запрошення Git Bash? {
=Unix-подібне середовище для роботи в Windows
~Назву віддаленого сервера GitHub
~Версію Python на комп'ютері
#### MINGW64 — «таке собі Unix-like середовище для роботи Windows», де ми працюємо з командним рядком.
}

::Копіювання та вставка в Git Bash::
Які комбінації клавіш для копіювання й вставки згадуються для Git Bash? {
=Ctrl+Insert для копіювання, Shift+Insert для вставки
~Ctrl+C і Ctrl+V як у звичайному Windows
~Alt+C і Alt+V
#### У Git Bash замість звичного Ctrl+C/Ctrl+V використовують Ctrl+Insert і Shift+Insert.
}

::Відкриття Git Bash з папки::
Як автор відкриває Git Bash безпосередньо в потрібній директорії? {
=Правою кнопкою миші → «Відкрити Git Bash тут» (або через контекстне меню)
~Тільки через PowerShell Admin
~Лише з меню Пуск без прив'язки до папки
#### Після встановлення в контекстному меню папки з'являється опція відкрити Git Bash, і поточна директорія відповідає обраній папці.
}

::Корисність офіційного сайту::
Навіщо автор радить зберегти git-scm.com у закладках? {
=Там є документація, посилання та книга про Git
~Там можна замовити фізичний сервер
~Сайт потрібен лише для одноразового завантаження
#### Окрім завантаження, на сайті «є корисні матеріали: документація, посилання, книга про Git» — вони знадобляться під час навчання.
}

::Інтерфейс Git Bash::
Як автор описує інтерфейс Git Bash? {
=Максимально мінімалістичний, з можливістю часткової кастомізації
~Повністю графічний, як у провіднику Windows
~Неможливо змінити шрифт чи тему
#### Інтерфейс «максимально мінімалістичний», але його «можна трошки вплинути» — теми, гарячі клавіші тощо.
}
