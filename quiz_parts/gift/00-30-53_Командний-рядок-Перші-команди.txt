::Метадані файлів::
Що таке метадані у контексті роботи з файлами? {
=Додаткова інформація про файл: тип, розмір, дата модифікації тощо
~Сам вміст тексту всередині файлу
~Назва операційної системи
#### Метадані «не повідомляють, що міститься в файлах», але додають інформацію: коли змінено, який тип, який розмір на диску.
}

::Команда ls::
Що робить команда ls? {
=Повертає вміст каталогу — список файлів і папок
~Видаляє поточну директорію
~Відкриває браузер з документацією
#### ls (list) повертає «вміст всього, що міститься в каталозі» — наприклад, folder2 і myfile.txt.
}

::Пробіли в назвах::
Чому автор радить уникати пробілів у назвах папок і файлів? {
=Пробіли часто викликають проблеми при зверненні до файлів у командному рядку
~Пробіли заборонені будь-якою операційною системою
~Пробіли автоматично видаляють вміст файлу
#### Папки з пробілами «надаються в лапках», і автор радить уникати пробілів — краще folder_2 або folder2.
}

::Поточний робочий каталог::
Що означає «поточний робочий каталог»? {
=Папка, у якій ми зараз перебуваємо і до якої за замовчуванням застосовуються команди
~Завжди корінь диска C
~Лише папка .git
#### За замовчуванням «усі наступні команди відносяться до поточного каталогу», якщо ми його не змінимо.
}

::Команда pwd::
Що робить команда pwd? {
=Виводить повний шлях до поточної директорії (print working directory)
~Копіює файл у нову папку
~Створює новий текстовий файл
#### pwd — print working directory: показує, де ми знаходимось, наприклад /c/Users/User/Desktop/folder1.
}

::Команда cd без аргументів::
Куди переводить команда cd без вказаного шляху? {
=У домашній каталог користувача
~У кореневий каталог диска C
~Видаляє поточну папку
#### Якщо просто виконати cd, ми «перейдемо в домашній каталог користувача».
}

::Дві крапки (..)::
Що означає синтаксис cd .. ? {
=Перехід на один рівень вище — до батьківського каталогу
~Видалення батьківської папки
~Відкриття прихованих файлів
#### «Дві крапки» дозволяють «зазирнути в батьківський каталог» або перейти на рівень вище.
}

::Команда clear::
Навіщо використовують команду clear? {
=Щоб очистити вікно термінала і зручніше продовжувати роботу
~Щоб видалити всі файли в папці
~Щоб скинути пароль користувача
#### clear повертає «чисте вікно», коли накопичується багато виводу — Git Bash при цьому пам'ятає історію команд.
}

::Історія команд::
Як у Git Bash повторити раніше введену команду? {
=Стрілками вгору та вниз
~Тільки заново набирати з нуля
~Командою history delete
#### «Навігація здійснюється завдяки стрілочкам вгору та вниз» — можна швидко повторити попередні команди.
}

::Команда mkdir::
Що робить mkdir (make directory)? {
=Створює нову директорію з указаною назвою
~Показує вміст усіх дисків
~Перейменовує файл
#### mkdir створює нову папку — наприклад, mkdir folder3 у поточному каталозі.
}
